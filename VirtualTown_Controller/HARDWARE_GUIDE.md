# VirtualTown — Hardware Bring-up & Test Guide (零基础版)

> 这份文档假设你 **从来没焊过电路**。我们会按"先点亮最便宜的，再驱动最贵的"
> 的顺序，**一个模块一个模块**地测试。每一步都有：
>
> 1. 接线图（文字版）
> 2. 一段 30 行左右的测试代码（在 `tests/` 文件夹里）
> 3. "看到什么 = 成功" 的验收标准
> 4. 常见故障 → 解决方法
>
> 全部走完，再回去烧 `VirtualTown_Controller.ino`，几乎一定能跑起来。

---

## 0. 安全与"绝对不要做"清单

> 这一节不是吓你，是因为下面这几条是新手最常烧板子的原因。

1. **不要用 Arduino 的 5V 引脚直接驱动直流电机或水泵。**
   Arduino 5V 输出最多 ~500 mA；R130 电机启动瞬间可能拉 ~1 A，水泵更高。
   后果：Mega 重启、USB 端口烧毁、甚至电脑 USB 控制器损坏。
   正确做法：**电机/水泵用独立 5–9 V 电源 + MOSFET/三极管开关**，见第 5 节。

2. **永远共地 (Common Ground)。**
   外部电源的 GND **必须** 和 Arduino 的 GND 用一根线连在一起，否则信号是悬浮的，
   MOSFET 不会正常导通。

3. **电机/水泵两端必须并联一个续流二极管 (1N4007)。**
   电机断电瞬间产生反向电压尖峰，会击穿 MOSFET 或三极管。

4. **接线/改线之前，先拔 USB 和外部电源。** 带电插拔最容易烧。

5. **LCD I2C 模块焊接的电位器（蓝色小方块）控制对比度。**
   买回来很多是拧到底的，屏幕一片漆黑或一片亮白属于正常，**不是坏的**——
   先调对比度（见第 1 节）。

6. **HX712 / MPM283 / DRV5055A1 在本作业的 VirtualTown 程序里 *用不到*。**
   如果你只是要完成"魔杖控制小镇"这个任务，可以**完全跳过**它们。
   但本指南最后一节给了它们的最小测试代码，方便你做别的实验。

## 1. 你还需要去买/找到的"配件"

主控、传感器、执行器你已经有了。下面这些**一定要再准备**，否则电路搭不起来：

| 配件 | 数量 | 作用 | 备注 |
|---|---|---|---|
| 面包板（830 孔，MB-102） | 1–2 块 | 免焊接搭电路 | 必备 |
| 杜邦线（公-公 / 公-母 各一包） | 一捆 | 连接 | 必备 |
| **N 沟道 MOSFET：IRLZ44N** *或* **NPN 三极管：TIP120** | 2 个 | 驱动风扇 + 水泵 | 二选一，**MOSFET 更省心** |
| **续流二极管 1N4007** | 2 个 | 保护 MOSFET | 必备 |
| 10 kΩ 电阻 | 2 个 | MOSFET 栅极下拉 | 必备 |
| 220 Ω 电阻 | 1–几个 | LED 限流 | 必备 |
| **5 V 1 A（或更大）独立电源**：USB 充电头 + USB 线剥皮，或 4 节 AA 电池盒 | 1 套 | 给电机/水泵供电 | **必备** |
| 万用表 | 1 块 | 测电压、查通断 | 强烈建议 |

> 如果你只能买一种功率开关，**选 IRLZ44N MOSFET**，它在 Arduino 5 V 直接驱动下
> 就能完全导通（"逻辑电平 MOSFET"），TIP120 在大电流下发热严重。

## 2. 总体电路架构（先看懂这张图再插线）

```
                    ┌────────────── USB (5V, 给逻辑用) ──────┐
                    │                                          │
                    │      ┌─────────────────────────┐         │
                    │      │     Arduino Mega 2560   │         │
                    │      │                         │         │
   IR receiver ─────┼──D11─┤                         │         │
   LED + 220Ω  ─────┼──D7──┤                         │         │
   MOSFET gate ─────┼──D5──┤  (PWM, 控风扇)          │         │
   MOSFET gate ─────┼──D6──┤  (开关, 控水泵)         │         │
   Buzzer +    ─────┼──D8──┤                         │         │
   LCD SDA     ─────┼──20──┤                         │         │
   LCD SCL     ─────┼──21──┤                         │         │
                    │      │   GND ──┬──────────────┐│         │
                    │      └─────────┼──────────────┘│         │
                    │                │                │         │
                    │           ★ 共地点 ★           │         │
                    │                │                │         │
                    │   ┌────────────┴───────┐        │         │
                    │   │  外部 5V 电源 (+)  ├─→ 给风扇/水泵正极
                    │   │  外部 5V 电源 (–)  │
                    │   └────────────────────┘
```

**记住三条总线**：
- **5 V 逻辑电** = USB 给的，给 LCD、IR、LED、MOSFET 栅极用。
- **5 V 功率电** = 外部电源给的，**只给电机和水泵用**。
- **GND 必须连在一起**（Arduino GND + 外部电源 GND）。

## 3. 软件准备

1. 安装 **Arduino IDE 2.x**：<https://www.arduino.cc/en/software>
2. 打开 IDE → **Tools → Board → Arduino Mega or Mega 2560**
3. **Tools → Port** 选择你插上去的那个 COM 口（Windows）/ `/dev/ttyACM0`（Linux/Mac）。
4. 装两个库：
   - **Sketch → Include Library → Manage Libraries…**
   - 搜 `IRremote` → 安装 **IRremote by shirriff, z3to** （**v4.x**）
   - 搜 `LiquidCrystal I2C` → 安装 **LiquidCrystal_I2C by Frank de Brabander**
5. 把本仓库的 `VirtualTown_Controller/` 文件夹整个用 IDE 打开。

## 4. 模块级 Bring-up 流程（**严格按顺序做**）

### 步骤 1 — I2C LCD 1602

**为什么先做它**：它是最便宜的、坏不了的、而且后面所有调试都靠它显示信息。

#### 接线
| LCD I2C 板 | Arduino Mega |
|---|---|
| `GND` | `GND` |
| `VCC` | `5V` |
| `SDA` | `20` (SDA) |
| `SCL` | `21` (SCL) |

> Uno 用户：SDA = A4，SCL = A5。

#### 测试 A：先扫描 I2C 地址
打开 `tests/01_i2c_scanner.ino` → 上传 → 打开串口监视器 (**右上角放大镜**)，
波特率选 **115200**。

**期望输出**：

```
I2C device found at 0x27   ← 或 0x3F
done
```

把这个地址记下来，如果不是 `0x27`，就回到 `VirtualTown_Controller.ino` 把
`LiquidCrystal_I2C lcd(0x27, 16, 2);` 改成你扫到的地址。

#### 测试 B：让屏幕显示字
上传 `tests/02_lcd_hello.ino`。

**期望**：第一行 `Hello, Town!`，第二行 `LCD OK :)`。

#### 看不到字怎么办？
- 屏幕全黑或全亮 → 用螺丝刀拧 LCD 背面那个**蓝色电位器**（对比度），慢慢拧到字出现。
- 背光不亮 → 检查 VCC/GND 是否反接。
- 串口扫不到 I2C 设备 → SDA/SCL 接错了；Mega 必须用 20/21，Uno 必须用 A4/A5。

---

### 步骤 2 — 城镇 LED

#### 接线
```
Arduino D7 ──[ 220Ω ]──┤▶├── GND
                       LED
                  长腿(+)  短腿(–)
```
> LED 长脚朝 D7 那侧（经过 220 Ω 电阻），短脚朝 GND。**不加电阻会烧 LED 和板子。**

#### 测试
上传 `tests/03_led_blink.ino`。

**期望**：LED 每秒亮灭一次。

#### 不亮怎么办？
- LED 装反了 → 调头试试。
- 没有 220 Ω → 千万**别**直接把 LED 接到 5V，会烧。
- 串口监视器里没看到 `LED ON / OFF` → COM 口/波特率不对。

---

### 步骤 3 — 蜂鸣器 / 喇叭

#### 接线（**无源**蜂鸣器或 8 Ω 喇叭）
| 元件 | Arduino |
|---|---|
| `+` (长脚) | `D8` |
| `–` (短脚) | `GND` |

> "无源"蜂鸣器需要 `tone()` 才会响；"有源"蜂鸣器一通电就嗡嗡叫。
> 本作业的旋律必须用**无源**蜂鸣器或小喇叭，否则只能听到一个固定频率。

#### 测试
上传 `tests/04_buzzer_tone.ino`。

**期望**：听到 do-re-mi-fa-sol 五个音，然后停 1 秒循环。

#### 没声音？
- 接反了 → 调头。
- 用的是有源蜂鸣器 → 只会"哔"一声不变调，换无源的。
- 喇叭功率太大（>1 W） → 中间串一个 100 Ω 限流，否则 Arduino 引脚电流不够。

---

### 步骤 4 — 红外接收器 VS1838B（**最关键**，决定遥控按键码）

VS1838B 正面（带凸起的圆球面）朝向你时，**从左到右**引脚是：

| VS1838B 引脚 | 含义 | 接到 Arduino |
|---|---|---|
| 1 (左) | `OUT` (信号) | `D11` |
| 2 (中) | `GND` | `GND` |
| 3 (右) | `VCC` | `5V` |

> **不同厂家顺序可能不同**！如果你的模块是焊在 PCB 小板上，板子上会标 `S/G/V` 或 `OUT/GND/VCC`。**先看丝印**。接反了不会立刻烧，但收不到信号。

#### 测试
上传 `tests/05_ir_dump.ino`。打开串口 (115200)，**对着接收器按遥控器**。

**期望**：

```
[IR] protocol=NEC, code=0xFD08F7, raw=0x......
[IR] protocol=NEC, code=0xFD8877, raw=0x......
...
```

按下数字键 `1`、`2`、`3`、`0` 和 `POWER`，把每个键打印出来的 **`raw=0x......`** 那个值抄下来。

> 之所以用 `raw`（即 `decodedRawData`），是因为 `VirtualTown_Controller.ino` 里
> 比较的就是 `IrReceiver.decodedIRData.decodedRawData`。两边必须用同一个字段。

记下 5 个值，回到主程序把这几行替换：
```cpp
constexpr uint32_t IR_KEY_1     = 0x........;   // 抄你按 1 时打印的 raw
constexpr uint32_t IR_KEY_2     = 0x........;
constexpr uint32_t IR_KEY_3     = 0x........;
constexpr uint32_t IR_KEY_0     = 0x........;
constexpr uint32_t IR_KEY_POWER = 0x........;
```

#### 收不到信号？
- 距离太近（<5 cm）反而可能丢码，退到 30 cm 试。
- 遥控器没电 → 用手机摄像头对着遥控器顶端按一下，能看到紫色光闪 = 有电。
- 接反了 → 按 V/G/S 顺序重接。
- 持续刷出 `0xFFFFFFFF` → 这是"按住不放的重复帧"，主程序会自动忽略，不影响。

---

### 步骤 5 — 风扇 (R130 直流电机) 用 MOSFET 驱动

> ⚠️ **本节是最容易烧板子的地方，请按图严格接线，先不通外部电源测一遍逻辑。**

#### 电路（用 IRLZ44N MOSFET）

IRLZ44N 正面（带散热片那一面对着你），从左到右脚是 **G (Gate) — D (Drain) — S (Source)**。

```
                    +5V (外部电源, 不是 Arduino 的 5V!)
                       │
                  ┌────┴────┐
                  │  风扇   │
                  └────┬────┘
                       │ (电机另一端)
            1N4007 ─ ▶│◀ ─        ← 二极管，黑色环 (阴极) 朝 +5V
                       │
                       ●  ← 这一点接 MOSFET 的 D (漏极)
                       │
              D ───────┤
                       │ MOSFET (IRLZ44N)
              G ───────┤   ← 这一点同时接：
              │        │     - Arduino D5  (PWM 信号)
              │        │     - 一个 10kΩ 下拉电阻到 GND
              S ───────┤
                       │
                      GND  ← 必须同时接 Arduino GND 和外部电源 GND
```

文字版接线表：

| 连接 | 起点 | 终点 |
|---|---|---|
| 风扇 + | 风扇红线 | 外部电源 +5V |
| 风扇 – | 风扇黑线 | MOSFET **D** |
| 续流二极管 | 风扇黑线 | 风扇红线（**带环的一头朝红线**） |
| MOSFET **S** | — | GND（公共地） |
| MOSFET **G** | — | Arduino **D5** |
| 10 kΩ 电阻 | MOSFET **G** | GND |
| **共地** | Arduino GND | 外部电源 GND |

#### 测试（**先不通外部电源**！）
1. 还**别**插外部电源。
2. 上传 `tests/06_motor_pwm.ino`。
3. 用万用表测 MOSFET 的 **G 对 GND**：
   - 程序进入 "ON" 阶段时应该 ≈ 5 V
   - 进入 "OFF" 阶段应该 ≈ 0 V
   - 进入 "PWM" 阶段应该是个中间值（万用表显示 ~2 V 左右是正常的）
4. **确认这一步无误后**，再插上外部电源。
5. 风扇应该按 `OFF → 慢 → 中 → 快 → OFF` 循环。

#### 风扇不转/狂转/Arduino 重启？
- Arduino 一通电就重启 → 你**用了 Arduino 5V 给电机供电**，赶紧改成外部电源。
- 风扇一直全速 → 可能 Gate 直接接 5V 了，或者忘了 10kΩ 下拉。
- 风扇完全不转 →
  - MOSFET 的 G/D/S 接反了；
  - 没有共地；
  - 用了非逻辑电平 MOSFET (如 IRF540)，5V 推不动，换 IRLZ44N。
- MOSFET 发烫 → 占空比太大 + 电流太大，先用更小的电机/更低 PWM 测。

---

### 步骤 6 — 水泵（电路和风扇**完全一样**，换个引脚）

把第 5 节的电路**复制一份**：

| 改动 | 值 |
|---|---|
| Arduino 引脚 | **D6** （不再是 D5） |
| MOSFET | 第 2 个 IRLZ44N |
| 二极管 | 第 2 个 1N4007 |
| 负载 | 水泵代替风扇 |
| 电源 | **可以和风扇共用** 同一组外部电源（确保电源 ≥1.5 A） |

#### 测试
1. **先把水泵放进装满水的小容器！** 干转 30 秒以上会烧绕组。
2. 上传 `tests/07_pump_test.ino`。
3. 水泵应该开 3 秒，停 3 秒，循环。

---

### 步骤 7 — 整合测试（终于可以跑主程序了）

确认 1–6 都通过后：

1. 把 `IR_KEY_*` 5 个常量替换成你在第 4 步抄下来的真值。
2. 如果 LCD 地址不是 `0x27`，改成你扫到的（例如 `0x3F`）。
3. 编译并上传 `VirtualTown_Controller.ino`。
4. 打开串口监视器 (115200)，应看到：
   ```
   [VirtualTown] Boot...
   [VirtualTown] IR ready on pin 11
   ```
5. LCD 第一行显示 `Virtual Town`，第二行 `Wave the wand!`。
6. 拿遥控器按按键：

| 按键 | 应该发生什么 |
|---|---|
| **1** | LED 亮，蜂鸣器"叮"一声，LCD 显示 `Good Morning, Town!` |
| **2** | 风扇转、水泵出水，LCD 显示 `Wind & Water On` |
| **3** | 风扇/水泵继续，蜂鸣器循环播放小调，LCD 显示 `Festival Mode!` |
| **0** 或 **POWER** | 全部停，LCD 显示 `Town is Sleeping...`，3 秒后清屏 |

## 5. 故障排除速查表

| 现象 | 最可能的原因 | 解决 |
|---|---|---|
| 上传时报 `avrdude: stk500v2_ReceiveMessage(): timeout` | COM 口选错；或者串口监视器没关 | 关掉串口监视器，重选 Port |
| LCD 一片黑 / 一片白 | 对比度电位器没调 | 拧 LCD 背面蓝色小方块 |
| LCD 完全不亮 | I2C 地址错；或 SDA/SCL 接反 | 跑 `01_i2c_scanner.ino` |
| 串口看到 IR 码全是 `0x0` | 红外接收器接反 | 换 V/G/S 顺序 |
| 串口看到的 IR 码每次都不一样 | 用错字段（用了 `command` 或 `address`） | 主程序和测试代码统一用 `decodedRawData` |
| 一插电 Arduino 就不停重启 | 电机/水泵从 Arduino 5V 取电 | 必须外部电源 |
| 风扇/水泵不转，但 MOSFET 摸着烫 | 接成了线性区 / 没有完全导通 | 换逻辑电平 MOSFET (IRLZ44N)，加 10kΩ 下拉 |
| 节日模式音乐时强时弱 / 卡顿 | 主循环里别的地方加了 `delay()` | 检查是否改了主程序，所有定时必须用 `millis()` |
| 灯亮但音乐不响 | 用了**有源**蜂鸣器 | 换无源蜂鸣器或 8Ω 喇叭 |

## 6. （选做）作业里其他元件的最小测试代码

下面这几个**本次 VirtualTown 程序不会用到**，但既然作业列出来了，给你最小验证程序，方便你单独玩。

### 6.1 DRV5055A1 线性霍尔传感器
- 引脚：1=VCC(3.3V/5V), 2=GND, 3=Vout (模拟)
- Vout 接 Arduino `A0`，VCC 接 5V，GND 接 GND
- 见 `tests/08_hall_drv5055.ino`，用磁铁靠近会看到电压偏离 ~2.5 V 中点。

### 6.2 MPM283 压力传感器 + HX712 ADC
- 这两个**必须配套**用，因为 MPM283 输出的是 mV 级差分信号，Arduino 自带 ADC 读不出来。
- HX712 接线：VCC=5V, GND=GND, DOUT=`A1`, SCK=`A2`，MPM283 的 +OUT/–OUT 接 HX712 的 INA+/INA–。
- 见 `tests/09_hx712_read.ino`（裸读 24 位原始值，给压力变化时数值会变）。

### 6.3 NE555 单稳态测试
- 这是纯模拟元件，不需要写代码。在面包板上按"R-C 触发器"接好后，
  按一下复位按钮，输出脚 (3 脚) 会保持高电平 `T = 1.1 × R × C` 秒。
- 用 `tests/10_ne555_pulse_count.ino` 让 Arduino D2 (中断) 数 555 输出的脉冲个数。
